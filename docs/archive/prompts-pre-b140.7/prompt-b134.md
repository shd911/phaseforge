## Что нужно сделать

### 1. Глобальные шорткаты

В `src/App.tsx` собрать единый keyboard listener (если ещё не централизован).
Активные комбинации (на macOS — `Cmd`, на Linux/Win — `Ctrl`):

| Шорткат | Действие | Источник |
|---|---|---|
| Mod+N | New Project | b131 (через FileMenu) |
| Mod+O | Open Project | b131 |
| Mod+S | Save | b131 |
| Mod+Shift+S | Save As | b131 |
| Mod+Z | Undo | b132 |
| Mod+Shift+Z | Redo | b132 |
| Mod+Shift+V | Открыть диалог «Версии…» | b134 (новое) |

Все шорткаты:
- **не** срабатывают когда фокус внутри `<input>`, `<textarea>`,
  `[contenteditable]`;
- проверяются через `event.metaKey || event.ctrlKey` (кросс-платформенно);
- НЕ конфликтуют между собой (особенно Mod+Z из b132 и Mod+S, Mod+Shift+Z
  и Mod+Shift+S — проверить порядок проверки, чтобы Shift+Z не ловился
  как Z);
- вызывают `event.preventDefault()` чтобы браузерный handler не сработал.

Если в текущем коде есть разрозненные listeners (в FileMenu, в стейте undo,
в SaveProject) — централизовать их в одном `globalShortcuts.ts` или прямо
в `App.tsx`. Все остальные точки удаляют свои локальные listeners.

### 3. Edge-case warnings

#### 3.1 Отсутствующие файлы измерений после restore

В `restoreState` (project-io.ts) уже логирует warnings в console при
`import_measurement` failure. Нужно собрать их в массив и после завершения
restore показать пользователю один итоговый toast:

> «Не найдены измерения: <band1>, <band2>. Полосы загружены без измерений,
>  переимпортируйте файлы вручную.»

Toast — через существующий механизм уведомлений в проекте (если такой есть).
Если нет — простой `<div>` с auto-dismiss через 8 секунд в правом нижнем углу.

#### 3.2 Подтверждение удаления версии

В VersionsDialog кнопка × — проверить что вызывает `confirm()` диалог
с текстом «Удалить версию "<description>"? Это действие нельзя отменить.»
(если b133 уже делает — оставить, если нет — добавить).

#### 3.3 Версия с пустым описанием при rebuild_index

Когда `rebuildSnapshotIndex()` восстанавливает индекс из orphan-файлов,
у которых не было `snapshot_description`, показать в списке
«(описание утеряно)» курсивом, чтобы пользователь понял что это не
обычная пустая запись.

#### 3.4 Auto-snapshot description при Restore

Сейчас формат предлагался: «Авто-снимок перед загрузкой '<desc>'».
Зафиксировать формат: `Авто-снимок · <дата HH:MM> · перед загрузкой "<desc>"`.
Это позволит отличать ручные версии от авто.

### 4. Индикаторы состояния в заголовке окна

`document.title` динамически:

- `PhaseForge — <projectName>` для сохранённого проекта.
- `PhaseForge — <projectName> •` если `isDirty=true` (звёздочка/буллет).
- `PhaseForge — Untitled` если проекта ещё нет.

Реализовать через `createEffect` в `App.tsx` на сигналах
`projectName`, `isDirty`.

### 5. Закрытие диалогов по Escape

`UnsavedChangesDialog`, `VersionsDialog`, `ProjectNameDialog` —
все должны закрываться по `Escape`. Если ещё не сделано — добавить.
Escape = эквивалент кнопки «Отмена».

### 6. Bump версии

- `src-tauri/tauri.conf.json` — version + productName/title до b134.
- `src-tauri/src/lib.rs` — startup-лог.
- После билда — skill `build-version`.
