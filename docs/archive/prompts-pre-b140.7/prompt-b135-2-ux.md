# Промт для Code: b135.2 — UX модалки анализа замера

**Тип:** UX hotfix. Bump до 0.1.0-b135.2.

## Контекст

Текущий диалог `MeasurementAnalysisDialog.tsx` имеет две параллельные
модели применения: чекбоксы + общая кнопка «Применить отмеченное», и
кнопка применения под каждым finding. Это запутывает: пользователь не
понимает какую использовать. После применения отдельной кнопкой блок
становится opacity 0.5 и воспринимается как «недоступный». Формулировка
«Игнорировать всё» вводит в заблуждение, когда что-то уже применено.

Цель: одна модель — кнопка под каждым finding применяет рекомендацию
сразу, плюс одна общая «Применить все» для пакетного действия.

## Что нужно сделать

### 1. `src/components/MeasurementAnalysisDialog.tsx`

**Убрать:**
- сигнал `selected` и функцию `toggleSelect`;
- чекбокс `<input type="checkbox">` у каждого finding;
- обёртку `<label>` вокруг блока finding (она была нужна для чекбокса);
- функцию `applySelected`;
- кнопку «Применить отмеченное».

**Оставить и адаптировать:**
- сигнал `appliedFindingIds` — отслеживание уже применённых;
- функцию `applyOne` — переименовать в `applyFinding` для ясности;
- кнопку под каждой рекомендацией (`<For each={f.recommendations}>`).

**Добавить:**
- функцию `applyAll`: для каждого finding из `req.result.findings`,
  если он ещё не в `appliedFindingIds`, применить его первую
  рекомендацию (`f.recommendations[0]`) через `applyRecommendation`,
  потом добавить в `appliedFindingIds`. После цикла — НЕ закрывать
  диалог автоматически (пользователь может посмотреть результат
  и закрыть сам).

**Кнопки внизу:**
- Слева: `[ Применить все ]` — disabled когда все findings уже applied
  (сравнить размеры `appliedFindingIds().size === findings().length`).
  Скрывается полностью когда `findings.length === 0`.
- Справа: `[ Закрыть ]` — всегда. Текст не меняется в зависимости от
  состояния. Никакого «Игнорировать всё».

### 2. Визуальные правки

**Применённый finding:** не блекнуть весь блок до 0.5. Вместо этого:
- заголовок остаётся читаемым (full opacity);
- кнопка под рекомендацией → disabled, текст меняется на
  `✓ <исходный label>` (например, было «Установить нижнюю границу
  = 35 Гц», стало «✓ Установлено: нижняя граница = 35 Гц»);
- мелкая зелёная пометка «применено» рядом с заголовком — оставить
  как сейчас.

Описание finding остаётся видимым, чтобы пользователь мог проверить
что именно было применено.

## Acceptance

1. Открыть диалог с findings — каждый finding имеет одну кнопку
   применения, **нет** чекбоксов.
2. Клик на кнопку под finding → рекомендация применена, кнопка
   становится disabled с префиксом «✓ Установлено: …», описание
   остаётся читаемым.
3. Кнопка «Применить все» внизу — применяет всё что ещё не applied,
   диалог остаётся открытым.
4. После применения всех findings кнопка «Применить все» становится
   disabled.
5. «Закрыть» закрывает диалог и ставит `analysis_dismissed = true`,
   независимо от того применено что-то или нет.
6. Cmd+Z после применения рекомендации — откатывает изменение
   (peqDirectLow возвращается к прежнему значению).
7. Чистый замер — модалка «Замер выглядит чисто», только кнопка
   «Закрыть» (b135.1 не сломан).
