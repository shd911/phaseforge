1. Замер (Measurement)

1.1 Загрузка файла

Пользователь загружает текстовый файл замера в формате REW (.frd / .txt). Файл содержит три столбца:

Частота (Гц) Амплитуда (дБ) Фаза (градусы, опционально)

Данные сохраняются в полосу (Band) как массивы: freq[], magnitude[], phase[].

1.2 Удаление задержки

Замер содержит задержку распространения звука (расстояние микрофон → динамик). Эта задержка проявляется как линейный наклон фазы. Программа автоматически находит и удаляет её:

  1. Метод 1 (при наличии sample rate): Вычисляет импульсную характеристику через обратное БПФ, находит пик — его позиция = задержка
  2. Метод 2 (робастный): Линейная регрессия фазы по частоте в полосе наилучшего отношения сигнал/шум. Наклон прямой = задержка
  3. Перекрёстная проверка: Если методы расходятся более чем на 50%, используется метод 2 (более устойчив для одиночных драйверов)

Удаление задержки:

фаза_новая[i] = фаза[i] - 360 × частота[i] × задержка_сек

После удаления проверяется «перекомпенсация» — если избыточная групповая задержка > 0.2 мс, выдаётся предупреждение.

1.3 Сглаживание

Амплитуда замера может быть сглажена перед отображением. Доступные режимы:

  • Variable (ERB): Ширина окна адаптируется к частоте (узкое на НЧ, широкое на ВЧ) — имитирует восприятие человеческого уха
  • Фиксированное: 1/3, 1/6, 1/12, 1/24 октавы — постоянная ширина в логарифмическом масштабе

Сглаживание влияет только на отображение, не на внутренние расчёты (PEQ, FIR и т.д. работают с исходными данными).

2. Таргет (Target)

Таргет — это желаемая АЧХ, к которой нужно привести динамик. Строится из нескольких компонентов, применяемых последовательно:

Базовый уровень (дБ) → Наклон (tilt) → ФВЧ (High-Pass) → ФНЧ (Low-Pass) → Низкочастотная полка → Высокочастотная полка

2.1 Базовый уровень

Начальная точка — горизонтальная прямая на заданном уровне (например, 0 дБ). Все остальные компоненты добавляются к ней.

2.2 Наклон (Tilt)

Линейный наклон в логарифмическом масштабе частоты. Задаётся в дБ/октаву относительно опорной частоты (обычно 1000 Гц).

амплитуда[i] += наклон × log2(частота[i] / опорная_частота)

Типичные значения: от -3 до +3 дБ/октаву. Используется для компенсации нарастания мощности к ВЧ в помещении.

2.3 Фильтры ВЧ и НЧ (High-Pass / Low-Pass)

Каждый фильтр вычисляет два массива:

  • Амплитуда (дБ) — всегда вычисляется, определяет крутизну среза
  • Фаза (градусы) — зависит от типа фильтра и флага linear_phase

Флаг linear_phase = true: фаза = 0° на всех частотах. Фильтр воздействует только на амплитуду, не вращая фазу.

Флаг linear_phase = false: фаза вычисляется из аналитической передаточной функции (зависит от типа фильтра). Для Gaussian — особый случай (см. раздел 9).

2.4 Полочные фильтры (Shelf)

Низкочастотная и высокочастотная полки добавляют усиление/ослабление ниже или выше заданной частоты. Реализованы как аналоговые фильтры 2-го порядка (формулы из RBJ Audio EQ Cookbook). Контролируются частотой, усилением (дБ) и добротностью Q.

2.5 Итоговый таргет

Все компоненты складываются последовательно в дБ (амплитуда) и градусах (фаза). На выходе — кривая, описывающая желаемый отклик:

Амплитуда_итого = базовый + наклон + ФВЧ + ФНЧ + полка_НЧ + полка_ВЧ Фаза_итого = 0 + 0 + фаза_ФВЧ + фаза_ФНЧ + фаза_полки_НЧ + фаза_полки_ВЧ

Фаза оборачивается в диапазон [-180°, +180°] для отображения.

Gaussian — подробнее

Gaussian фильтр определён на линейной шкале частот:

H(f) = exp(-ln(2) × (f / fc) ^ (2M))

Где M — коэффициент формы (1..10). На частоте среза fc: H = 0.5 (−6 дБ), как у Linkwitz-Riley.

Ключевое свойство: при M ≥ 2 сумма LP + HP = 1.0 (идеально плоская). Это делает Gaussian идеальным для кроссоверов.

Проблема: Для Gaussian нет аналитической формулы фазы в частотной области. Поэтому:

  • В режиме linear_phase: фаза = 0° (только амплитуда)
  • В режиме min-phase: фаза вычисляется через преобразование Гильберта от амплитуды

4. Параметрический эквалайзер (PEQ)

PEQ — набор параметрических полос, каждая задаётся тремя параметрами:

  • Частота (Гц) — центральная частота воздействия
  • Усиление (дБ) — подъём (+) или срез (−)
  • Добротность Q — ширина воздействия (выше Q = уже полоса)

Каждая полоса реализована как цифровой биквадный фильтр 2-го порядка (IIR). Для каждой частоты замера вычисляется комплексная передаточная функция — из неё извлекаются амплитуда (дБ) и фаза (градусы).

Результаты всех полос складываются (в дБ для амплитуды, в градусах для фазы).

Автоматическая оптимизация PEQ

Алгоритм Левенберга-Марквардта (LMA) автоматически подбирает параметры PEQ-полос, минимизируя разницу между замером и таргетом:

  1. Инициализация: грубый подбор полос по наибольшим отклонениям
  2. Итеративная оптимизация: одновременная подстройка всех параметров (f, G, Q)
  3. Ограничения: частота в пределах полосы, усиление ограничено, Q ∈ [0.5, 10]

Весовая функция: Более чувствительная в полосе 800–5000 Гц (зона максимальной чувствительности слуха), менее чувствительная выше LP-фильтра, нулевой вес на глубоких провалах (акустические нули не корректируются).

5. Скорректированная кривая (Corrected)

Corrected = Замер + PEQ + Кроссовер. Показывает, как будет выглядеть отклик после коррекции.

Замер (mag, phase) → + PEQ (mag, phase) → + Кроссовер (mag, phase) = Corrected

Все сложения — в дБ для амплитуды и в градусах для фазы (эквивалент умножения комплексных спектров).

5.1 Кроссовер (Cross-Section)

Функция cross-section вычисляет вклад только фильтров ВЧ/НЧ (без наклона, полок и базового уровня). Возвращает амплитуду и фазу фильтров.

Отличие от таргета:

Таргет (evaluate_target)Кроссовер (cross-section)
Все компоненты: наклон + полки + фильтрыТолько ФВЧ + ФНЧ
Абсолютный уровень (с базовым)Относительный (добавляется к замеру)
Для отображения таргетной кривойДля расчёта corrected = замер + кроссовер

5.2 Проблема Gaussian в Corrected

Текущее ограничение: Cross-section для Gaussian фильтров возвращает фазу = 0°, т.к. аналитическая фаза отсутствует. Это означает, что кривая Corrected не учитывает фазовый вклад Gaussian кроссовера в режиме min-phase.

На вкладке Target эта проблема решена — используется преобразование Гильберта. Но в corrected path на вкладках IR/Step фаза кроссовера Gaussian = 0, что приводит к некорректной позиции импульса и степа.

6. Отображение по вкладкам

6.1 SPL / Freq — Амплитудно-частотная характеристика

Левая ось Y: Амплитуда (дБ). Правая ось Y: Фаза (градусы, [-180°, +180°]).

Кривые:

  • Measurement — замер (сплошная) + фаза замера (штриховая)
  • Target — таргет (сплошная) + фаза таргета (штриховая)
  • Corrected — скорректированная (сплошная) + фаза скорректированной (штриховая)

Все фазовые кривые обёрнуты в [-180°, +180°].

6.2 IR / Step — Импульсная и переходная характеристики

Преобразование из частотной области во временну́ю через обратное БПФ:

Амплитуда + Фаза → Комплексный спектр → ОБПФ (IFFT) → Импульс → Кумулятивная сумма = Step

Алгоритм:

  1. Выбор размера БПФ: N = max(4 × кол-во точек, 4096), округлённый до степени 2
  2. Интерполяция амплитуды/фазы на линейную сетку частот (0 Гц → Найквист)
  3. Построение комплексного спектра: H[k] = 10^(mag/20) × e^(j×phase)
  4. Зеркальное отражение для сопряжённой симметрии (отрицательные частоты)
  5. ОБПФ → импульс во временной области
  6. Переупорядочивание: пик импульса выравнивается на t=0
  7. Нормализация: пик = 100%
  8. Step = кумулятивная сумма импульса, нормализованная по пику

Три кривые: Measurement IR/Step, Target IR/Step, Corrected IR/Step.

6.3 GD — Групповая задержка

Групповая задержка — производная фазы по частоте (с обратным знаком):

GD(f) = -(1 / 360) × d(фаза) / d(частота) [секунды]

Вычисляется численно (центральная разность). Фаза предварительно разворачивается (unwrap), чтобы избежать скачков на ±180°.

Три кривые: GD замера, GD таргета, GD скорректированной.

6.4 Export — Модель vs FIR

Вкладка Export показывает сравнение идеальной модели и реализованного FIR-фильтра:

  • Model dB / Model ° — целевая амплитуда и фаза (таргет + PEQ, нормализованные)
  • FIR dB / FIR ° — реализованная АЧХ и фаза FIR-фильтра

Метрики: количество отсчётов (taps), частота дискретизации, тип окна, каузальность, pre-ringing, ошибка амплитуды, пульсации групповой задержки.

7. Режим суммирования (SUM)

В SUM-режиме несколько полос (Band) суммируются когерентно — с учётом фазы, как в реальной акустической системе.

Алгоритм когерентного суммирования:

  1. Для каждой полосы: нормализация амплитуды по пику (все полосы вносят равный вклад)
  2. Перевод в прямоугольные координаты:
    Re[i] += A × cos(phase) Im[i] += A × sin(phase) где A = 10^((mag + offset) / 20)
  3. Накопление Re и Im по всем полосам
  4. Обратный перевод в амплитуду/фазу:
    mag_sum = 20 × log10(sqrt(Re² + Im²)) phase_sum = atan2(Im, Re) × 180/π
  5. Вычисление IR/Step из суммарного спектра через ОБПФ

Суммируются отдельно: Σ замеров, Σ таргетов, Σ скорректированных.

8. Генерация FIR-фильтра

FIR (Finite Impulse Response) — конечный импульсный отклик, экспортируемый как WAV-файл для применения в DSP-процессоре.

Таргет (mag+phase) → + PEQ → Модель → FIR engine → Импульс → WAV

8.1 Пайплайн FIR

  1. Формирование целевой коррекции:
    • В полосе пропускания (между ФВЧ и ФНЧ): используется таргет
    • За пределами полосы: плавный переход к сглаженному замеру (сигмоидный бленд ±0.5 октавы)
  2. Коррекция = таргет − замер (в дБ)
  3. Интерполяция на линейную сетку БПФ (0 → Найквист)
  4. Ограничения:
    • Максимальный подъём (max_boost_db) — ограничивает пиковую коррекцию
    • Шумовой порог (noise_floor_db) — обрезает коррекцию в шуме
    • Узкополосное ограничение — сглаживание 1/3 октавы, ограничение превышения над сглаженной кривой
  5. Фаза:
    • Min-phase: Гильберт от логарифма амплитуды коррекции → минимально-фазовая
    • Linear-phase: Фаза = 0 (симметричный импульс)
    • Hybrid: Коррекция (min-phase) + форма фильтра (linear-phase)
  6. ОБПФ → временно́й импульс
  7. Оконная функция: Hann / Blackman / Kaiser — подавляет артефакты обрезки
  8. Итеративное уточнение: До 10 итераций — измерение отклонения реализованного FIR от цели, компенсация
  9. Нормализация: Пик реализованного спектра = 0 дБ

8.2 Режимы фазы FIR

Minimum-Phase

  • Каузальный (нет pre-ringing)
  • Полуокно (правая часть симметричного)
  • Минимальная задержка
  • Фаза зависит от амплитуды коррекции

Linear-Phase

  • Симметричный импульс (pre-ringing = post-ringing)
  • Полное окно
  • Постоянная задержка = N/2 отсчётов
  • Нулевая фазовая ошибка

9. Особая обработка Gaussian

Gaussian — единственный тип фильтра, для которого нет аналитической формулы фазы. Это требует особой обработки в разных частях программы:

Вычисление таргетаФаза = 0°Фаза = 0° (аналитической нет)
Отображение фазы таргета0°Гильберт от амплитуды таргета
Cross-section (corrected)0°0° (проблема!)
FIR генерацияLinearPhase modeMinimumPhase mode + Гильберт
WAV экспортLinearPhaseMinimumPhase + Гильберт

Известное ограничение (b99): Corrected IR/Step не получает фазу Gaussian min-phase от кроссовера, т.к. cross-section возвращает 0°. Это влияет на позицию и форму corrected step для басовых полос с Gaussian кроссовером.

Преобразование Гильберта

Вычисляет минимально-фазовую характеристику из амплитуды:

  1. Интерполяция амплитуды на линейную сетку БПФ
  2. Ограничение динамического диапазона (80 дБ)
  3. Вычисление ln(амплитуда_линейная)
  4. БПФ → применение окна Гильберта (×2 для положительных частот, ×0 для отрицательных) → ОБПФ
  5. Мнимая часть результата = минимально-фазовая характеристика (радианы)
  6. Обратная интерполяция на логарифмическую сетку, перевод в градусы
